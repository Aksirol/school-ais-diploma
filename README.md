# АІС «Навчальний процес» (Гімназія 5-9 класи)

Автоматизована інформаційна система для управління навчальним процесом у гімназії. Проєкт розроблений як дипломна робота, що охоплює повний цикл взаємодії між адміністрацією, педагогами та учнями.

## 🚀 Основні модулі
- **🔐 Авторизація та безпека**: Багаторівневий доступ (Admin, Teacher, Student) на базі JWT.
- **📊 Дашборд аналітики**: Візуалізація успішності за допомогою графіків Recharts.
- **📝 Журнал оцінок**: Електронний журнал з автоматичним розрахунком середнього бала.
- **📂 Навчальні матеріали**: Система завантаження та зберігання конспектів, презентацій та відео (з перевіркою MIME-типів).
- **💬 Месенджер**: Чат у реальному часі на Socket.IO з індикацією статусів (Онлайн/Офлайн) та прочитання.
- **📅 Розклад та Відвідуваність**: Керування графіком занять та цифровий облік присутності.
- **⚙️ Адмін-панель**: Управління користувачами, класами та навчальними призначеннями.

## 🛠 Технологічний стек
- **Backend:** Node.js, Express, TypeScript, PostgreSQL (Sequelize), Socket.io, Multer, Bcryptjs.
- **Frontend:** React 18, TypeScript, Vite, Tailwind CSS, Recharts, Lucide-react, Axios.
- **Infrastructure:** Docker (Postgres, pgAdmin).

## 📦 Встановлення та запуск

### 1. Підготовка бази даних (Docker)
Переконайтеся, що у вас встановлено Docker, та запустіть контейнери з кореня проєкту:
```bash
docker-compose up -d
```

### 2. Налаштування Backend
Перейдіть у папку сервера, встановіть залежності та запустіть сервер:
```bash
cd backend
npm install
npm run dev
```

Для створення першого адміністратора (виконайте в окремому терміналі):
```bash
npm run seed:admin
```

### 3. Налаштування Frontend
Перейдіть у папку клієнта та запустіть сервер розробки:
```bash
cd frontend
npm install
npm run dev
```

## 🏗 Архітектура системи
Система побудована за принципом розділення відповідальності (Separation of Concerns). Бекенд реалізує RESTful API та WebSocket-сервер для миттєвої передачі даних. Фронтенд використовує контекстну модель (Context API) для управління станом авторизації та захисту маршрутів.

## 🔒 Безпека
- Перевірка прав доступу на рівні об'єктів (IDOR protection).
- Валідація вхідних даних на стороні сервера (Express Validator).
- Безпечне завантаження файлів з жорстким обмеженням форматів та розміру.
